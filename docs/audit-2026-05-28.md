# Asahi laptop setup — audit report

**Date:** 2026-05-28
**Scope:** `~/dotfiles` (Hyprland, waybar, scripts, `system/` privileged tree),
`~/src/hypr-aerial`, live session state, installed privileged files.
**Method:** static review of tracked + untracked files, running-process
inspection, dependency presence, privilege/secret/temp-file surface checks.

Severity key: **CRIT** (exploitable or breaks the session) · **HIGH** (live
functional bug) · **MED** (security posture / latent breakage) · **LOW**
(hardening / hygiene).

---

## Findings

### F1 — Trackpad gesture daemon started twice  · HIGH · FIXED
`config/hypr/hyprland.conf` had `exec-once = …/hypr-trackpad-gestures.py` on
**both line 31 and line 32**, so two copies of the daemon run at login
(`pgrep -fc` reported 2). Two daemons each read the same trackpad and each fire
the notification-tray toggle, so a single right-edge swipe toggles the panel
**twice** → it appears to do nothing. This is the root cause of the
"gestures are inconsistent / don't work" reports.
**Fix:** removed the duplicate line; killed the stray running instance.

### F2 — Passwordless sudo for all of `wheel`  · MED · accepted
`system/etc/sudoers.d/wheel-nopasswd` grants `wheel ALL=(ALL) NOPASSWD: ALL`.
Any process running as your user can become root with no prompt — so any
compromised script/daemon (several run from `~/dotfiles` and from the AUR-ish
copr packages) is a full-system compromise, silently. This is a deliberate
single-user-laptop tradeoff and is recorded in memory; **left as-is** but
flagged. If you ever want to tighten it, scope NOPASSWD to the specific
commands you actually run unattended (brightnessctl, systemctl suspend, the
login-manager helpers) instead of `ALL`.

### F3 — greetd greeter stack is broken and would leak the password  · MED · latent
Not currently deployed (no active display-manager symlink, `greetd` inactive,
nothing installed in `/usr/local`), so this is latent — but if you run
`install-hyprland-system-files.sh` it ships a **non-functional and insecure**
login path:
- `system/usr/local/bin/hypr-greeter-app` ends with
  `exec hypr-greeter-login "$PROMPT_USER" "$password"` — **`hypr-greeter-login`
  does not exist anywhere** in the repo or on disk, so the greeter exits
  non-zero on every login attempt → you'd be locked out at the greeter.
- It passes the password as a **command-line argument**, which is visible in
  `/proc/<pid>/cmdline` to any local process for the lifetime of the exec.
**Recommendation:** do not deploy the greetd stack until the helper is written
to read the password from stdin (not argv) and actually exists. Tracked as
follow-up, not fixed in this pass (it's inert today).

### F4 — Large amount of unversioned work  · MED · partially fixed
`git status` showed one modified file and a pile of untracked files that were
never committed, including the **entire privileged `system/` tree** (sudoers,
polkit, PAM, root sbin scripts) and ~8 scripts + 2 docs. Privileged,
hand-edited code living only in the working tree is a data-loss and
"what-actually-got-deployed" risk.
**Fix:** committing the genuinely-in-use scripts/docs and the modified
notification panel. The `system/` greetd tree is **intentionally left
untracked pending F3** so a broken login path can't be `git checkout`-restored
and deployed by accident.

### F5 — `XDG_RUNTIME_DIR or /tmp` fallback in daemons  · LOW
`waybar-sysmon.py`, `notification-panel.py`, `waybar-hover-refresh.py`,
`liquid-osd.py` and others fall back to a fixed `/tmp/...` path when
`XDG_RUNTIME_DIR` is unset. `/tmp` is world-writable, so the fallback path is
predictable and another user could pre-create/symlink it. In a single-user
session `XDG_RUNTIME_DIR` is always set (`/run/user/1000`), so this never
triggers in practice. Hardening only: fall back to
`/tmp/<name>-$(id -u)` with `0700`, as `liquid-osd.py` and `glass_shader`
already did. Not fixed (cosmetic on a single-user box).

### F6 — `notification-panel` clear-all history drain  · LOW · acceptable
`on_clear` loops `restore` + `dismiss --no-history` up to 50× to empty mako's
history (mako has no clear-history verb). Each restore briefly re-displays a
notification; on a very full history this can flicker. Capped, so it
terminates. Acceptable.

### F7 — Patched Aquamarine is a local build  · LOW · maintenance
`/usr/lib64/libaquamarine.so.0.11.0` is your local fix for the Asahi
render-node bug. A `dnf upgrade aquamarine` from the copr will overwrite it
silently and (if the shipped version still lacks the fix) drop you back to
llvmpipe software rendering for every client. Re-verify after any
Hyprland/aquamarine upgrade. (Already in memory; restated here for the audit.)

### F8 — Gesture + hover daemons depend on `input` group + AT-SPI  · LOW · note
`hypr-trackpad-gestures.py` needs the user in the `input` group to read
`/dev/input/event*`; `waybar-hover-refresh.py` and `workspace-osd.py` depend on
AT-SPI being up. These are started from `exec-once` but have no restart-on-crash
supervision — if one dies, it's gone until next login. Consider moving them to
user systemd units with `Restart=on-failure` if you want them resilient.
