## Critical and High Fixes Applied

### 1. Firewalld was disabled while SSH was exposed

Evidence before fix:

- `firewalld`: disabled and inactive.
- `ss -tulpen`: `sshd.service` listening on `0.0.0.0:22` and `[::]:22`.
- Other LAN-visible listeners included mDNS and LLMNR.
- Saved firewalld default zone was `FedoraWorkstation`, which allowed broad
  high TCP/UDP ports.

Fix:

```sh
sudo firewall-offline-cmd --set-default-zone=public
sudo systemctl enable --now firewalld
```

Evidence after fix:

- `firewall-cmd --state`: `running`
- default zone: `public`
- active zone: `public` on `wld0`
- allowed services: `dhcpv6-client mdns ssh`
- no broad high-port allow rules.

### 2. Greetd greeter would fail login

`system/usr/local/bin/hypr-greeter-app` had a latent runtime bug: `do_auth()`
required `password`, `start_cmd`, and `start_env`, but the login thread called
it with only `password`. A real greetd login would fail before starting the
session.

Fix:

- Thread now passes selected session command/env into `do_auth()`.
- Password entry is cleared immediately after reading.
- greetd session cleanup is guarded if socket/session creation fails.
- Session selector added with Hyprland default plus installed fallback sessions.
- Fixed file was installed to `/usr/local/bin/hypr-greeter-app`.

Validation:

```sh
python3 -m py_compile system/usr/local/bin/hypr-greeter-app
sudo /home/ben/dotfiles/scripts/install-hyprland-system-files.sh --check
```

### 7. AI Waybar modules were made responsive and idempotent

Fixes:

- Codex usage now renders the local cache immediately and refreshes in the
  background, instead of blocking the Waybar item on network/account work.
- Codex Waybar polling is 30 seconds; Claude is 60 seconds.

## Remaining Risks and Recommendations

### Passwordless sudo remains enabled

Current sudo policy includes:

```text
ben ALL=(ALL) NOPASSWD: ALL
%wheel ALL=(ALL) NOPASSWD: ALL
/etc/sudoers.d/99-ben-nopasswd
```

This is convenient for local automation but high trust. I did not remove it
because prior local setup and this audit workflow depend on noninteractive sudo.
Recommended hardening is to replace broad NOPASSWD with a narrow allowlist.

### SSH password authentication remains enabled

`sshd -T` reports:

```text
passwordauthentication yes
pubkeyauthentication yes
permitrootlogin prohibit-password
```

Firewall now limits exposure to allowed zone policy, but password SSH is still
weaker than key-only SSH. Since `~/.ssh/authorized_keys` exists, the next
hardening step is to disable password auth after confirming every expected
client can log in by key.

### Reboot required

DNF reports core libraries/services updated since boot. Reboot is needed to
fully apply the update set.

### Tailscale DNS warning

`tailscale netcheck` reported:

```text
Tailscale can't reach the configured DNS servers. Internet connectivity may be affected.
```

UDP connectivity otherwise looked functional. This should be checked after
reboot and after confirming Tailscale DNS settings.

### XDPH screencopy token default

`config/hypr/xdph.conf` still has:

```text
allow_token_by_default = true
```

This was previously documented as intentional for the local setup. For a
stricter privacy posture, switch it off and test screen sharing.

### Journal noise

Recurring boot/session errors remain, mostly Asahi/Broadcom Wi-Fi P2P scan
errors, Bluetooth disconnect noise, duplicate notification DBus service names,
and earlier `hyprlock` crashes. No failed services remained after the update.
